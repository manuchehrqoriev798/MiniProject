# Deploy Exam Platform (easiest way)

One app: **frontend + backend** run together. The backend serves the React build and the API from a single URL. No CORS setup needed in production.

---

## Option 1: Render (recommended – free tier, GitHub)

1. **Push your repo to GitHub** (org or your account).

2. **Render**
   - Go to [render.com](https://render.com) → Sign in with GitHub.
   - **New → Blueprint**.
   - Connect the repo and select it.
   - Render will read `render.yaml` and create:
     - A **Web Service** (Docker: builds frontend + runs FastAPI).
     - A **PostgreSQL** database and `DATABASE_URL` for the app.

3. **First deploy**
   - Deploy runs: build Docker image (React build + backend), run `alembic upgrade head`, start the app.
   - Your app URL will be like: `https://exam-platform.onrender.com` (or the name you gave the service).

4. **Environment**
   - In the Web Service → **Environment**, set **CORS_ORIGINS** to your app URL (e.g. `https://exam-platform.onrender.com`) if you use a different frontend URL later.
   - **SECRET_KEY** is auto-generated by the blueprint; you can replace it with your own if you want.

5. **Done.**  
   Open the app URL: you get the React UI; API is at the same origin (e.g. `/api/auth/login`).

---

## Option 2: Deploy without Blueprint (Render manual)

1. In Render: **New → Web Service**.
2. Connect the GitHub repo.
3. Set:
   - **Runtime**: Docker.
   - **Dockerfile path**: `./Dockerfile` (root of repo).
   - **Instance type**: Free (if available).
4. **New → PostgreSQL** (free), then in the Web Service → **Environment** add:
   - **DATABASE_URL** → “Internal Database URL” from the Postgres service.
   - **SECRET_KEY** → generate a long random string.
   - **CORS_ORIGINS** → your Render app URL (e.g. `https://your-service.onrender.com`).
5. **Advanced** → **Release Command**: `alembic upgrade head`.
6. Deploy.

---

## Option 3: Railway

1. Go to [railway.app](https://railway.app) → Login with GitHub.
2. **New Project → Deploy from GitHub repo** → choose this repo.
3. **Add PostgreSQL** (from Railway’s data tab).
4. In your Web Service:
   - Set **Root Directory** to repo root (so it finds `Dockerfile`).
   - It should detect Docker and use the Dockerfile.
   - Add variables: **DATABASE_URL** (from Postgres), **SECRET_KEY**, **CORS_ORIGINS** (your Railway app URL).
5. Add a **Deploy** trigger or run migrations once:  
   `railway run alembic upgrade head` (or run the same in a one-off shell in Railway).

---

## Env vars summary

| Variable        | Required | Example / note |
|----------------|----------|----------------|
| `DATABASE_URL` | Yes      | `postgresql://user:pass@host:5432/dbname` (from Render/Railway Postgres). |
| `SECRET_KEY`   | Yes      | Long random string (JWT signing). |
| `CORS_ORIGINS` | Optional | Your app URL if frontend is on another domain; same-origin deploy can leave default. |

## Run the same setup locally (Docker)

```bash
# Build and run (needs Docker, and a Postgres or DATABASE_URL)
docker build -t exam-platform .
docker run -p 8000:8000 -e DATABASE_URL="postgresql://..." -e SECRET_KEY=dev-secret exam-platform
```

Then open `http://localhost:8000`.

## Two separate deploys (frontend and backend)

If you prefer **frontend** and **backend** on different URLs:

- **Backend**: Deploy the backend only (e.g. Render Web Service with Dockerfile that only runs FastAPI, or a `Dockerfile.backend` without the frontend stage). Set **CORS_ORIGINS** to your frontend URL.
- **Frontend**: Deploy the React app (e.g. Vercel, Netlify, or Render Static Site):
  - Build command: `cd frontend && npm ci && npm run build`
  - Publish directory: `frontend/build`
  - Env: **REACT_APP_API_URL** = your backend URL + `/api` (e.g. `https://your-api.onrender.com/api`).

The single-app setup above is simpler (one URL, no CORS).
